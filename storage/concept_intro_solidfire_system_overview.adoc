= SolidFire -Speichersystem
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
Ein SolidFire All-Flash-Speichersystem besteht aus einzelnen Hardwarekomponenten (Laufwerken und Knoten), die durch die NetApp Element Software, die unabhängig auf jedem Knoten läuft, zu einem einzigen Pool von Speicherressourcen kombiniert werden.  Dieser einheitliche Cluster stellt sich externen Clients als ein einziges Speichersystem dar und wird über die Element-Software-Benutzeroberfläche, die API und andere Verwaltungstools als eine einzige Einheit verwaltet.

Mithilfe der Benutzeroberfläche der NetApp Element Software können Sie die Speicherkapazität und Leistung des SolidFire Clusters einrichten und überwachen sowie die Speicheraktivität in einer Multi-Tenant-Infrastruktur verwalten.

image::../media/solidfire_concepts_architecture_image.gif[Architekturdiagramm der Element-Software]

Ein SolidFire All-Flash-Speichersystem umfasst folgende Komponenten:

* Knoten: Physische Hardware, die die Speicherressourcen für einen Cluster bereitstellt.  Es gibt zwei Arten von Knoten:
+
** Speicherknoten: Server, die eine Sammlung von Laufwerken enthalten.
** Fibre Channel (FC)-Knoten: Werden verwendet, um FC-Clients über einen Fibre Channel-Switch zu verbinden.


* Cluster: Das Herzstück des SolidFire -Speichersystems, bestehend aus mindestens vier Knoten.
* Management-Knoten: Ermöglicht die Aktualisierung und Bereitstellung von Systemdiensten einschließlich Überwachung und Telemetrie, die Verwaltung von Cluster-Assets und -Einstellungen, die Ausführung von Systemtests und -dienstprogrammen sowie den Zugriff auf den NetApp -Support zur Fehlerbehebung.  Der Management-Knoten (mNode) ist eine virtuelle Maschine, die parallel zu einem Element-Software-basierten Speichercluster läuft.
* Active IQ: Ein webbasiertes Tool, das kontinuierlich aktualisierte historische Ansichten von clusterweiten Daten bereitstellt.  Sie können Benachrichtigungen für bestimmte Ereignisse, Schwellenwerte oder Kennzahlen einrichten.  Active IQ ermöglicht es Ihnen, die Systemleistung und -kapazität zu überwachen und sich über den Zustand des Clusters auf dem Laufenden zu halten.
* In den Speicherknoten werden Laufwerke verwendet, die Daten für den Cluster speichern.  Ein Speicherknoten enthält zwei Arten von Laufwerken:
+
** Volume-Metadaten speichern Informationen, die die Volumes und andere Objekte innerhalb eines Clusters definieren.
** Blocklaufwerke speichern Datenblöcke für Anwendungsvolumes.
